PART 5 – Training, testing, and final validation with YOLOv5

The purpose of Part 5 is to train the YOLOv5 model on the prepared custom dataset, test the trained model, record the final detection results, and prepare the final outputs for the project report. At this stage, all technical preparation from Part 4 must already be completed.

Part 5 starts only after:
- Parts 1–3 are completed
- the final dataset is placed in the correct train and validation folders
- the dataset has been verified successfully
- the YOLOv5 environment is ready

Main folders and files used in Part 5:
- ~/YOLOv5
- ~/YOLOv5/dataset/images/train
- ~/YOLOv5/dataset/images/val
- ~/YOLOv5/dataset/labels/train
- ~/YOLOv5/dataset/labels/val
- ~/YOLOv5/yolov5_data/custom_dataset.yaml
- ~/YOLOv5/scripts/verify_dataset.py
- ~/yolov5
- ~/yolov5/data/custom_dataset.yaml
- ~/yolov5/train.py
- ~/yolov5/detect.py
- ~/yolov5/runs/train
- ~/yolov5/runs/detect

What must be done in Part 5:

1. Activate the prepared YOLOv5 environment
Before training, activate the Python virtual environment:

source ~/yolov5_env/bin/activate

This environment already contains:
- Python
- pip
- installed requirements
- PyTorch
- CUDA support
- GPU access

2. Make sure the final dataset is ready
Before training starts, check that:
- all final images are already placed in train and val folders
- all final label files are already placed in train and val folders
- each image has a matching YOLO .txt label file
- the verification script has already been run successfully

Verification command:
python ~/YOLOv5/scripts/verify_dataset.py

3. Copy the dataset YAML file into the YOLOv5 framework
The dataset configuration file prepared in the project repository must be copied into the YOLOv5 framework folder:

cp ~/YOLOv5/yolov5_data/custom_dataset.yaml ~/yolov5/data/custom_dataset.yaml

This file tells YOLOv5:
- where the dataset is located
- which folders contain train and validation images
- how many classes are used
- what the class names are

4. Run the training
Go into the YOLOv5 framework folder and start training:

cd ~/yolov5
python train.py --img 640 --batch 16 --epochs 30 --data data/custom_dataset.yaml --weights yolov5s.pt --cache

This is the default training command for the project.

Meaning of the main parameters:
- --img 640 = image size used during training
- --batch 16 = number of images processed together
- --epochs 30 = number of full training cycles
- --data data/custom_dataset.yaml = dataset configuration
- --weights yolov5s.pt = starting pretrained YOLOv5 model
- --cache = speed up training by caching images

The default model used in this project is:
- yolov5s.pt

5. Use CPU fallback only if needed
If GPU training is not available, use the fallback command:

cd ~/yolov5
python train.py --img 416 --batch 4 --epochs 10 --data data/custom_dataset.yaml --weights yolov5n.pt --device cpu

The CPU fallback is slower and uses a smaller model:
- yolov5n.pt

6. Resume training if interrupted
If training stops or is interrupted, resume it with:

cd ~/yolov5
python train.py --weights runs/train/exp/weights/last.pt --resume

This continues training from the last saved checkpoint.

7. Check the training results
After training starts, YOLOv5 creates output folders under:
~/yolov5/runs/train/

The most important files are usually:
- best.pt
- last.pt

These are stored in:
runs/train/exp/weights/

best.pt is usually used for final testing and detection.

8. Run detection on images
After training, test the model on validation images:

cd ~/yolov5
python detect.py --weights runs/train/exp/weights/best.pt --img 640 --conf 0.25 --source ~/YOLOv5/dataset/images/val

This generates prediction results and saves them in:
~/yolov5/runs/detect/

9. Run detection on webcam if needed
To test the model on a live webcam stream, use:

cd ~/yolov5
python detect.py --weights runs/train/exp/weights/best.pt --source 0 --conf-thres 0.4 --img 640 --device 0

This can be used to demonstrate real-time object detection for the final presentation or report.

10. Record the final video
A short video showing object detection must be recorded.
The video should clearly demonstrate that the model can detect:
- screwdriver
- wrench

It is recommended to test:
- screwdriver only
- wrench only
- both objects together

11. Save the final outputs
At the end of Part 5, the following results should exist:
- trained YOLOv5 model weights
- detection output images or videos
- final video link
- dataset link if needed
- updated report with final results

12. Update the final report
After training and testing, the report must be completed with:
- final dataset information
- final train/val split
- short training description
- test results
- video link
- photo or dataset link
